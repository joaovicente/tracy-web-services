Tracy Web Services REST API

This service is used by Tracy Web Application
Tracy Web Services key domain objects are:
  • Applications: Allow grouping a set of Tasks logically
  • Task: Captures a specific (measurable) interface (i.e. a service endpoint), which can be further filterable by Tracy
There is a user REST service that supports the following operations
  • GET v1/applications/{application}/tasks/{task}/measurement - obtains performance measurement for a Task
  • GET v1/applications/{application}/measurement - obtains performance measurement for a all Tasks associated with an Application
  • GET v1/applications/{application}/tasks/{task}/analysis - obtains performance analysis for Task
The view operations are HTTP GET, and update is using HTTP PUT. From a web browser you can access the first two services using the following links
  • applications/SimulatedApp/measurement - to measure Application
  • applications/SimulatedApp/tasks/StaticExcellentTask/measurement - to measure Task
  • applications/SimulatedBatchApp/tasks/BatchExcellentTask/analysis - to analyse Task

This service offers an API browser using Swagger which is accessible from the following link

  • api-doc - Api Doc

To use the swagger ui, follow this link. Replace the URL value with this one to access the REST local resources http://localhost:8080/tws/v1/api-doc

  • swagger - swagger ui page

This service is based on Apache Camel. If you want to learn more about Camel REST see camel-example-swagger-cdi and camel-example-swagger-java
