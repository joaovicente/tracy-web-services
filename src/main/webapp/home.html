Apache Camel REST sample service using Swagger

This example shows how to use Servlet REST to define REST endpoints in Camel routes using the Rest DSL, and using camel-swagger-java to expose the REST service APIs.
This sample project is based on the example projects provided in the Camel repository
It differs from the examples in that, it provides a single example that:
  • Bundles a WAR containing swagger-ui
  • Uses the Java Rest DSL
  • Is self contained (i.e. does not depend from Camel examples parent pom)

There is a user REST service that supports the following operations

  • GET /user/{id} - to view a user with the given id
  • GET /user/final - to view all users
  • PUT /user - to update/create an user
The view operations are HTTP GET, and update is using HTTP PUT. From a web browser you can access the first two services using the following links
  • user/123 - to view the user with id 123
  • user/findAll - to list all users
From the command shell you can use curl to access the service as shown below: 
    curl -X GET -H "Accept: application/json" http://localhost:8080/camel-swagger-svc/v1/user/123
    curl -X GET -H "Accept: application/json" http://localhost:8080/camel-swagger-svc/v1/user/findAll
    curl -X PUT -d "{ \"id\": 666, \"name\": \"The devil\"}" -H "Accept: application/json" http://localhost:8080/camel-swagger-svc/v1/user
Notice when trying to get user with id 789 then there is a built-in delay of 0.5-2 seconds, to simulate some slowness in the service.

This example offers an API browser using Swagger which is accessible from the following link

  • api-doc - Api Doc

To use the swagger ui, follow this link. Replace the URL value with this one to access the REST local resources http://localhost:8080/camel-swagger-svc/v1/api-doc

  • swagger - swagger ui page
